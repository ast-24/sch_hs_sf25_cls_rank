タイマー機能拡張について

API
- エンドポイントに /progress を追加
    - GET /progress で /ready と /timemng の情報をまとめて返す(リクエスト数削減用)

- エンドポイントに /progress/ready を追加
    - POST /progress/ready で準備完了をセット
        - 部屋番号を送信する
        - DBに保存
    - DELETE /progress/ready で準備完了を解除
        - DBのフラグをクリア
        - 成功したら true を返す
    - GET /progress/ready で現在の準備完了状態を返す
        - 各部屋が準備完了かどうかのフラグを返す

- エンドポイントに /progress/timemng を追加
    - POST /progress/timemng でタイマー開始時刻をセット
        - JSTで次の次の N分M0秒 にセットしDBに保存
        - 例: 現在時刻が 14:02:08 の場合、14:02:20が開始時刻となる
        - 設定された時刻を返す
        - タイマー継続時間も受け取っておく(この段階ではロジックに使用しない)
    - DELETE /progress/timemng でタイマーを中止しDBの時刻をクリア
    - GET /progress/timemng で次のタイマー開始時刻を返す
        - タイマーがセットされていない場合は null を返す
        - タイマーがセットされている場合はその開始時刻と継続時間を返す

FE
- staff以下のページに timemng ページを追加
    - /progress/readyをポーリング(10秒)して各部屋の準備完了状態を表示
    - 時間を選択(デフォルト3分 分と秒でも入力可能に)
    - timer開始ボタンを追加
        - 押すと /progress/timemng へPOSTリクエストを送る
    - timer中止ボタンを追加
        - 押すと /progress/timemng へDELETEリクエストを送る
    - timer開始ボタンを押すとサーバから帰ってきた開始時刻を表示しその時刻までのカウントダウンを表示、
        そしてその時刻になったら選択された時間後までのカウントダウンを開始
    - 終了したらストップボタンだけ表示し、押すと最初の画面に戻る

- staffのenterページに準備完了/未完了ボタンを表示
    - 押すと /progress/ready へPOSTリクエストを送る
    - 押すと /progress/ready へDELETEリクエストを送る

- staffのenterページにタイマーを表示
    - 10秒ごとに /progress/timemng へGETリクエストを送り、次のタイマー開始時刻を取得
    - タイマー開始時刻とそこまでのカウントダウンを表示(ラウンド開始ボタンの直上)
    - 開始時刻になって継続時間内ならラウンド中という表示とともに残りのラウンド時間を表示
    - ※勝手に開始処理はしない
