タイマー機能拡張について

API
- エンドポイントに /progress/ready を追加
    - POST /progress/ready で準備完了をセット
        - 部屋番号を送信する
        - DBに保存
    - DELETE /progress/ready で準備完了を解除
        - DBのフラグをクリア
        - 成功したら true を返す
    - GET /progress/ready で現在の準備完了状態を返す
        - 各部屋が準備完了かどうかのフラグを返す

- エンドポイントに /progress/timemng を追加
    - POST /progress/timemng でタイマー開始時刻をセット
        - JSTで(次の A時 B分 C*10秒) + 10秒後 にセットしDBに保存
        - 例: 現在時刻が 14:02:08 の場合、14:02:20が開始時刻となる
        - 設定された時刻を返す
        - タイマー継続時間も受け取っておく(この段階ではロジックに使用しない)
        - 各部屋の準備状態をクリア
    - DELETE /progress/timemng でタイマーを中止しDBの時刻をクリア
    - GET /progress/timemng で次のタイマー開始時刻を返す
        - タイマーがセットされていない場合は null を返す
        - タイマーがセットされている場合はその開始時刻と継続時間を返す

- latestroomsのランキングエンドポイントを修正
    - (タイマー開始時刻+タイマー継続時間-30秒)以降に終了したラウンドのみを最新ラウンドとして扱いフィルタリングする
    - ただしタイマーがセットされていない場合は(5分ルール適用後の)全てのラウンドを対象とする
    - 既存の5分ルールも併用する
    - ※キャッシュ更新では5分ルールを使用し、getで返す際に更にフィルタリングする
        - headは最終更新時刻が(タイマー開始時刻+タイマー継続時間-30秒)より前なら現在を変更時刻とする
    - もしフロントエンドでlatestroomを更にフィルタリングしている場合はそちらのコードは削除

FE
- staff以下のページに timemng ページを追加
    - /progress/readyをポーリング(10秒)して各部屋の準備完了状態を表示
    - 時間を選択(デフォルト3分 分と秒を入力可能に)
    - timer開始ボタンを追加
        - 押すと /progress/timemng へPOSTリクエストを送る
        - 現在押した場合の予測開始時刻を表示(ローカルで計算)
    - timer中止ボタンを追加
        - 押すと /progress/timemng へDELETEリクエストを送る
    - timer開始ボタンを押すとサーバから帰ってきた開始時刻を表示しその時刻までのカウントダウンを表示、
        そしてその時刻になったら選択された時間後までのカウントダウンを開始
        - 中止ボタンも用意
    - 終了したらストップボタンだけ表示し、押すと最初の画面に戻る(ここではAPIにはリクエストを送らない)

- staffのenterページに準備完了/未完了ボタンを表示
    - 押すと /progress/ready へPOSTリクエストを送る
    - 押すと /progress/ready へDELETEリクエストを送る
    - ※押せる条件は"ラウンド開始"と同じ

- staffのenterページにタイマーを表示
    - 10秒ごとに /progress/timemng へGETリクエストを送り、次のタイマー開始時刻を取得
    - タイマー開始時刻とそこまでのカウントダウンを表示(ラウンド開始ボタンの直上)
    - 開始時刻になって継続時間内ならラウンド中という表示とともに残りのラウンド時間を表示
    - ※勝手に開始処理はしない

- staffのenterのinroundページにタイマーを表示
    - 10秒ごとに /progress/timemng へGETリクエストを送り、次のタイマー開始時刻と継続時間を取得
    - タイマー開始時刻とそこまでのカウントダウンを表示
    - 開始時刻になって継続時間内ならラウンド中という表示とともに残りのラウンド時間を表示
    - タイマーが終了済みならそう表示する
    - ※勝手に終了処理はしない

- ルートに/timeページを追加
    - 上段に現在時刻+100年後の日時をリアルタイムで表示(年から秒まで)
        ※これはカウントダウンタイマとは無関係で、装飾目的
    - 下段にタイマー
    - タイマーがセットされていて開始時間前の場合は"開始まで"でその開始時間までのカウントダウンを表示
      ※5秒前から1秒ごとにカウントダウンする 6秒前以前はもうすぐ開始します的な表示
    - タイマーがセットされていて開始時間後の場合は"残り"で終了時間までのカウントダウンを表示
    - 終了したら終了という表示に変わる
    - 10秒後にクリアする(初期状態とこの状態では何も表示しない(デザイン的に))

DB
- 部屋番号と準備状態を持つテーブルを追加(複数レコード)
- タイマーの開始時刻と継続時間を持つテーブルを追加(1レコード)
